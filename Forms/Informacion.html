Franklin Alejandro López Moncada Pedido_Proveedor El formulario Pedido_Proveedor posee 5 APIs que dentro de ellas permiten las siguientes funciones:
• Cargar los pedidos
• Agregar uno nuevo
• Modificar uno existente
• Eliminar algún registro
• Cargar los pedidos
Para la primera opción (Cargar los pedidos) utilizo la función CargarPedidos_Proveedor, de la cual hace un GET a la base de datos y permite mostrar los registros que hay en ella. Para la segunda opción (Agregar uno nuevo), está la función AgregaPedido_Proveedor, la cual mediante un formulario podremos agregar un registro totalmente nuevo a la base de datos del cual está conectado. Para la tercera opción (Modificar uno existente), está la función ActualizarPedido_Proveedor que está se activa mediante el botón Editar para podes hacer uso de ella, lo que conlleva a la modificación del registro seleccionado, modificándolo en el mismo formulario utilizado para agregar un registro nuevo. Y como última opción (Eliminar algún registro) tenemos la función EliminarPedido_Proveedor, que permite eliminar un registro seleccionado específicamente. Como última anotación, también tenemos la función CargarPedido_Proveedor, que es una función que permite cargar el registro seleccionado, al momento de querer editarlo.
José Manuel Durón Llanes Facturas El formulario Facturas_cliente contiene las 5 APIS :
1- Cargar Facturas
2- Cargar Factura en especifico
3- Actualiza una Factura
4- Eliminar una Factura
5- Agregar una Factura
Para la primera API lo que hace es obtener todos los registros que tengamos en nuestra base de datos y lo muestra de manera descendiente, la segunda api carga un registro en especifico de nuestra BD, la tercera Actualiza un registro que previamente nosotros hayamos seleccionado, la cuarta elimina una factura que hayamos seleccionado y la ultima API agrega datos a nuestra BD y los guarda.
Cheyla Carolina Solórzano Flores Socios de Negocio
1. Carga de registros de Socio: Se utilizó la función CargarSocios, de tipo GET que permite visualizar todos los registros almacenados en la Base de Datos.
2. Agregar nuevo socio: Se utilizo la función AgregarSocio, de tipo POST que permite agregar un socio nuevo a la Base de Datos.
3. Actualizar Socio: Se utilizo la función ActualizarSocio, de tipo PUT, se habilita un botón “Editar” que permite la modificación del registro y por siguiente agregarse a la lista de registros.
4. Carga de un único registro: Se utillizo la función CargaSocio, en el cual se carga el registro seleccionado y luego editarlo.
5. Eliminar Registro: Se utilizo la función EliminarSocio, de tipo DELETE, permite eliminar el registro seleccionado.
Evelin Magdalena Ponce Salgado Materiales El formulario Materiales, contiene 5 Apis que tienen las siguientes funciones:
1. Cargar materiales
2. Cargar Material
3. Insertar Material
4. Actualizar Material
5. Eliminar Material
La primera se utiliza la función CargarMateriales, donde se hace uso del método GET para que se muestren todos los registros que hay en la base de datos. En la segunda se utiliza la función CargarMaterial que se carga un material en específico, que este dentro de la base de datos. La tercera se trata de insertar un nuevo material con la función AgregarMateriales, donde mediante el formulario podemos agregar un nuevo registro a la base de datos. Luego cree la función ActualizarMaterial, donde podemos actualizar o modificar uno de los registros ya creados en la base de datos. Por ultimo hice la creación de la función Eliminar Material que consiste en eliminar registros que estén dentro de la base de datos.
